refactor(vbap): SLERP interpolation, improved fallbacks, and render resolution modes

This commit introduces significant improvements to the VBAP spatial audio renderer
for better accuracy and reliability in spatial trajectory rendering.

## Summary of Changes

### 1. SLERP Interpolation for Directions
- Replaced linear Cartesian interpolation with spherical linear interpolation (SLERP)
- Prevents near-zero direction vectors when interpolating between opposing keyframes
- All keyframe directions are normalized before interpolation
- Uses `safeNormalize()` helper to avoid division by zero

### 2. Improved Fallback Logic
- Changed fallback behavior when time is outside keyframe bounds
- Now uses nearest keyframe direction instead of global front (0,1,0)
- Before first keyframe: use first keyframe's direction
- After last keyframe: use last keyframe's direction
- Provides contextually appropriate fallback for each source

### 3. Deprecated Smooth Mode
- `smooth` render resolution now prints deprecation warning
- Gain interpolation in smooth mode can cause audible artifacts
- Recommended alternative: `block` mode with small block size (e.g., 64)

### 4. Block Center Sampling
- Direction is now sampled at block center: `(blockStart + blockLen/2) / sr`
- Reduces aliasing compared to sampling at block start
- More representative of the block's average direction

### 5. Explicit timeUnit JSON Field
- Added `timeUnit` field to spatial trajectory JSON schema
- Supported values: "seconds" (default), "samples", "milliseconds"
- Replaces heuristic-based time unit detection
- Maintains backward compatibility (defaults to seconds)

### 6. Per-Source Diagnostics
- Added logging for silent/missing sources at end of render
- Reports sources with all-zero audio or missing trajectory data
- Helps identify pipeline issues early

### 7. Documentation Updates
- Updated RENDERING.md with new CLI options, algorithms, and recommendations
- Created json_schema_info.md documenting JSON schemas
- Updated README.md with VBAP renderer options section

## Breaking Changes
None - all changes are backward compatible. The `smooth` mode still works but
prints a deprecation warning.

## Default Configuration
- Render Resolution: `block`
- Block Size: `64` samples
- Time Unit: `seconds` (if not specified in JSON)
